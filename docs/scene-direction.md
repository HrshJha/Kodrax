### This is the order I'd actually build and maintain them.

The **Scene Direction** document defines the artistic vision.

The **Motion Design System** defines how that vision moves.

The **Interaction System** defines how users affect it.

The **Component Specification** defines how engineers implement it.

That separation keeps each document focused, avoids duplication, and gives you documentation that's close to what you'd see on a high-end product or design team.
